数组的概念

数组:

一组具有相同数据类型元素的集合

就像一条绿箭口香糖,里面放的都是一片片的口香糖.
数组元素的特点

1 数据类型完全一致

2 数据在内存中是物理连续的

优点:数组会将存储的数据自动编号,省却了创建了多个变量的工作

数组的使用

数组的创建

两种方式
方式1:

int array1[] = {1,2,3,4,5};
int[] array2 = {6,7,8,9,0};

方式2:

int array1[] = new int[6];
int[] array2 = new int[6];

数组的访问和修改

arr[2] = 12;

这里注意一下,重点体现出数据位置的偏移,指出下标的索引是从0开始的

集合框架

*为什么使用集合框架?

集合框架就是一种容器,使用数组有一些不方便,数组长度不可变,没有方法,链表的缺点则刚好相反,而集合框架就是把两者的优点都集合起来,这就是集合框架.

Collection

Collection是一个集合接口,是集合对象的父接口.规定了集合对象的功能.集合对象可以随意的添加删除对象.

Collection下分两个接口,

List : 数据是有序的,和数组链表类似,可以存放重复的数据

Set : 数据是无序的,没有顺序的概念,数据的获取是采用的别的方式,不可以存放重复数据

我们在这两个里面分别讲一个常用的.
List下讲ArrayList,Set下讲一个HashMap

ArrayList

常用方法:

  • 1 添加

    list.add();

  • 2 获取元素

    list.get(1);

  • 3 获取索引

list.indexOf();

  • 4 移除元素

list.remove();

HashMap

HashMap是一种类似字典的键值对存储数值方式.

常用方法:

  • 添加方法:
    map.put(“key”,”value”);

  • 取值方法
    map.get(“key”);

  • 一处键值对
    map.remove(“key”);

  • 判断一个key是否已经存在
    map.containsKey(“key”);

  • 判断一个value是否已经存在
    map.containsValue(“value”);

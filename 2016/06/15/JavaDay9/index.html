异常

异常就是导致程序中断的错误.如果不对异常进行正确的处理,可能导致程序中断.

注意 : 异常不是错误!!

在Java中,异常也是一个对象.

异常分为两类,一类是检查异常,必须人为干预,否则编译不通过,一类是运行时异常,不是必须人为干预,特点是都是继承自RuntimeException.

异常的处理

  • 不进行处理直接throw出去,交给调用方进行处理

  • 使用try .. catch 进行接收处理

try..catch 处理

try中检测到异常会将异常对象传递给catch,catch捕获到异常进行处理.

try .. catch .. finally 处理

不论异常是否出现,都会执行finally模块里面的代码.

总结:

1 try必须出现。catch和finally至少出现一个

2 try是执行的代码 catch是捕获相应的异常后进行的处理，finally是无论如何都会进行的执行

3 代码中必须有这种异常才可以去catch，否则会提示不可到达的异常错误

4 不确定的时候可以直接catch Exception，但是要把位置放在其他的catch异常之后，否则会出错的

5 finally一般都是用来进行一些释放资源的

6 打印堆栈信息 e.printstacktrace

7 try与finally的选择标准：能自行处理就不要throws出去

8 捕获多个异常可挨个罗列所有的异常

9 子类不能比父类throws更多或者更大的的异常

throw

作用 : 是用来自己扔出去一个异常
语法： throw new RuntimeException（”throw动作”）

当你catch到一个异常之后第一件事就是打印堆栈信息，再好点儿就是记录日志，再好点就是修复异常。
异常记录的内容，谁，在什么时候，调用了什么方法，出现了什么信息，出现了什么错误

IO

IO就是输入输出 input output
文件的输入输出 内存的输入输出

File类

File : 用来描述文件或是一个目录,在windows下叫文件夹
层级分隔符linux下的是/而在windows下是\
大部分的应用都是File(“路径”);file创建支持相对路径。
用法: File file = new File(“C:\1.txt”);这样file就相当于是1.txt的抽象表述

常用方法

File类有几个常用的方法:

  • ListRoot();
  • canRead（）;
  • canWrite();
  • isHidden();
  • canExcute();//可执行的
  • Exists();
  • creatNewFile();
  • getAbsoluteFile();获取绝对路径;
  • isFile();
  • isDirectory();
  • lastModified():

如果要在之前不存在的文件夹下创建一个文件的话Java是不允许的，要先执行一遍file.makedirs()方法才可以。

IO

输入输出流
字节流 字符流
byte char
当读取二进制文件，当读取网络上的资源，一律使用字节流，当你确定要读的文件时纯文本的时候才会使用字符流去读。

当看到的是后缀是Stream的一定是字节流，后缀是Writer的是字符流，字节流的祖先是InputStream和OutputStream。字符流的祖先是Reader和Writer
在内存中使用的管道式ByteArrayInputStream，在文件中的使用的是管道FileInputStream，网络中使用的是直接用socket提供的。

使用方法：FileInputStream fis = new FileInputStream(file);
int i = fis.read();
read读完返回-1;
